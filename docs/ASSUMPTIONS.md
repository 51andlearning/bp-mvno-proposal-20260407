# Assumptions

## 1. No Internal Data Access

This proposal site is built entirely from information provided in the source proposal document (`BP MVNO PWA build and USSD proposal - MVNE 20260331.docx`) and the accompanying `proposal.md`. No access to internal BP systems, databases, or live platform data is assumed or required.

## 2. Directional Strategy Only

Pricing, timelines, and scope described on this site are directional and based on the proposal as issued. They represent MVNE's quoted position at time of issue and do not constitute a binding contract until a comprehensive SOW is signed.

## 3. Requires Validation Post-Approval

All commercial figures (once-off fees, monthly retainers, payment milestones) must be validated against the final signed SOW before being treated as contractual. Pricing is subject to USD/ZAR exchange rate fluctuations.

## 4. Proposal Validity Window

The proposal is valid for **30 days** from date of issue. The site should be treated as a living document — content may need to be updated if a new proposal version is issued.

## 5. No Authentication Required

The site is publicly accessible via its Vercel URL. It does not require login or authentication. If confidentiality is a concern, a password-protected Vercel deployment or access restriction can be configured post-launch.

## 6. No CMS

Content is managed via static files in `src/content/`. There is no headless CMS integration at this stage.

## 7. Single Language

The site is English-only. No i18n setup is included.
